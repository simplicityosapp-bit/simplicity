## חלק ב׳ — איך התשתית עובדת (לעיון)

**המנגנון הקנוני:**
```js
// כל מחרוזת מופנית־למשתמש:
const { addr } = useAddress()
addr({ male: 'תוכל', female: 'תוכלי', neutral: 'תוכל/י' })

// דאטה ממוגדרת (תפקידים, שאלות): resolver ייעודי
roleLabel('therapist', gender)   // מטפל / מטפלת / מטפל/ת
qtext('focus', gender)           // כמה ממוקד/ת/ממוקד הרגשת היום?
```
- מקור המגדר: `prefs.design.gender` ('male'|'female'|'neutral'). באונבורדינג שלב 1 הוא state מקומי חי (עוד לא נשמר), ולכן שם קוראים ל-`addressUser(localGender, …)` ישירות.
- **תאימות לאחור:** השמטת המגדר → צורת נייטרלי (לוכסן). לכן כל קוד ישן שלא הועבר עדיין ממשיך להציג בדיוק כמו היום — אין סיכון שבירה.
- **מוכן לרב־לשוניות:** המבנה (מילון variants + resolver) הוא בדיוק נקודת הכניסה לתרגום עתידי — מחליפים את ה-resolver במנגנון i18n בלי לגעת ב-call-sites.

## חלק ג׳ — ⚠️ החלטה מרכזית שצריך לקבל קודם: מהו "קול" המערכת?

הסקירה חשפה שהאפליקציה מערבבת **שלושה משלבים** של פנייה — וזה צריך הכרעה אחת שתנחה את כל ההמרה:

1. **נייטרלי בלוכסן** — `בחר/י`, `נסה/י שוב` (רוב המסכים).
2. **רבים** — `ברוכים הבאים`, `הוסיפו לקוח`, `הגדירו יעד` (מצבי ריק וברכות בית).
3. **יחיד זכר נעול** — `כל מה שתמחק`, `אתה בדרך הנכונה` (באגים — שגוי לנשים).

**השאלה:** האם רוצים (א) **התאמת מגדר מלאה** דרך `addr()` בכל מקום (זכר/נקבה אמיתי לפי הבחירה) — או (ב) להשאיר **רבים** כקול־בית ניטרלי במקומות מסוימים (ברכות/מצבי ריק) ולמגדר רק פנייה ישירה? 

ההמלצה שלי: **(א) התאמה מלאה** בכל מקום שמדבר אל המשתמש ביחיד, ולשמור "רבים" רק לכותרות שיווקיות/חגיגיות (כמו ה-HomeWelcome). ההכרעה הזו קובעת איך נמיר את ~88 המחרוזות שלמטה.

## חלק ז׳ — הצעות ותובנות

- **התשתית כבר הייתה — ויתומה.** מישהו בנה את `address.js` (`addressUser` + מילון `ADDRESS`) אבל לא חיבר אותו לשום מקום. כלומר הכוונה למגדור קיימת בקוד; רק לא הושלמה. זה מסביר את ה"חצי־מצב" של לוכסנים בכל מקום.
- **דפוס ה-resolver (qtext/roleLabel) הוא הזהב לרב־לשוניות.** כשנגיע ל-i18n, כל מחרוזת שכבר עברה ל-`addr()`/resolver היא נקודת תרגום מוכנה. שווה, תוך כדי מעבר הקופי, **לרכז מחרוזות למילון אחד** (במקום inline) — זה הופך את התרגום העתידי לטריוויאלי. אפשר לשקול קובץ `lib/strings.he.js` שבהדרגה יבלע את כל הטקסטים.
- **באגי המגדר־הנעול הם תסמין.** "תזיני" (נקבה נעול ב-Step8 — תוקן) ו"אתה בדרך הנכונה" (זכר נעול — תוקן) מצביעים שהקופי נכתב לאורך זמן ע"י אנשים שונים בלי כלל אחיד. התשתית החדשה מונעת הישנות.
- **טרמינולוגיה לפני תרגום.** כדאי לסגור את כפלי המונחים (פגישה/מפגש, כסף/כספים, נטו/מאזן) **לפני** שמתרגמים — אחרת מתרגמים את הבלבול לשפות נוספות.
- **מצבי ריק הם הזדמנות קופי.** רבים מהם ("עדיין אין…") הם הנקודות שבהן משתמש חדש לומד את המוצר — שווה מעבר ממוקד עליהם בנפרד.
